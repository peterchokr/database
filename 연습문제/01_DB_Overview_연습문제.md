### 13번 파일 시스템의 문제점

**모범 답안** (3가지 예시):
1. **수정 이상**: 학생 이름이 여러 파일에 저장되어 모두 수정해야 함 → 일관성 문제 발생
2. **삭제 이상**: 마지막 수강 강좌 삭제 시 학생 정보도 함께 삭제됨 → 필요한 데이터 손실
3. **삽입 이상**: 강좌가 없는 학생은 데이터 입력 불가능 → 새 학생 등록 어려움

---

### 14번 데이터 보안의 중요성

**모범 답안** (실생활 예시):
- **병원**: 의사만 환자 의료 기록 조회 가능 (환자 프라이버시 보호)
- **은행**: 직원은 계좌주인 계좌만 볼 수 있음 (자산 정보 보호)
- **학교**: 학생은 자신의 성적만 조회 가능 (개인 정보 보호)
- **결론**: DBMS는 이를 자동으로 제어하므로 매우 중요함 (법적 준수, 신뢰성 확보)

---

### 15번 정규화 vs 비정규화 비교

**모범 답안**:

**정규화 데이터베이스**
- 장점: 데이터 중복 제거, 일관성 보장, 저장공간 절감
- 단점: JOIN으로 인한 성능 저하 가능

**정규화되지 않은 데이터베이스**
- 장점: 빠른 조회 성능 (JOIN 없음)
- 단점: 데이터 중복, 수정 시 일관성 문제

**언제 정규화를 우선**
- 데이터 정확성이 중요한 업무
- 예: 금융(은행), 의료(병원), 행정(정부)

**언제 비정규화를 고려**
- 읽기가 많고 속도가 중요한 경우
- 예: 분석, 보고서, 대량 조회

### 18번 온라인 서점 데이터베이스 설계

**모범 답안**:

**1. 필요한 테이블 수**
- 4개의 테이블 (책, 고객, 주문, 주문상세)

**2. 각 테이블 간의 관계**
```
고객 (1) ──────→ (N) 주문
                   │
                   ↓
주문상세 (N) ←────┘
    │
    └────→ (1) 책
```
- 고객 1명이 여러 주문을 가짐 (1:N)
- 주문 1개가 여러 상품을 포함 (1:N)
- 책 1권이 여러 주문에 포함될 수 있음 (1:N)

**3. 정규화가 필요한 이유**
- 책 정보의 중복 제거 (같은 책을 여러 주문에서 반복 저장하지 않음)
- 책의 가격 변동 시 한 곳만 수정하면 됨 (일관성 보장)
- 저장 공간 절감

### 20번 테이블 설계 개선

**모범 답안**:

**1. 문제점의 명칭 (이상 현상의 종류)**
- **반복 그룹 (Repeating Group)**: 학생 정보가 강좌 수만큼 반복됨
- **수정 이상 (Modification Anomaly)**: 교수 이름 변경 시 모든 기록에서 수정 필요
- **삽입 이상 (Insertion Anomaly)**: 강좌 없는 학생은 등록 불가능

**2. 개선된 재설계 (정규화)**

원본 테이블 (문제 있음):
```
student_info (student_id, name, email, phone, course_id, course_name, professor_name, grade, semester)
```

개선된 설계 (4개 테이블):
```
student (student_id, name, email, phone)
course (course_id, course_name, professor_id)
professor (professor_id, professor_name)
enrollment (student_id, course_id, grade, semester)
```

관계도:
```
student (1) ────────→ (N) enrollment ←────── (1) course
                          │                      │
                          └─ grade, semester     │
                                              professor
```

**3. 개선된 설계의 장점**
- **데이터 중복 제거**: 학생 정보가 한 번만 저장됨
- **일관성 보장**: 교수 이름 수정 시 professor 테이블만 수정하면 됨
- **삽입 용이**: 강좌 없는 학생도 student 테이블에 바로 등록 가능
- **저장 공간 절감**: 반복되는 정보 제거로 효율성 증대
- **유지보수 간편**: 각 테이블이 명확한 목적을 가짐
